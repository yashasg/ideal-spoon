## Output Summary

- **Position:** QLoRA quantizes only the frozen base; adapters, gradients, optimizer state remain bf16/8-bit.
- **Measured gap:** Within run noise for DAPT/CPT and translation SFT; real ceilings are LoRA rank, corpus, tokenizer, LR schedule.
- **Failure modes that exist:** Outlier-channel-heavy bases under NF4, embedding/lm_head training on extended vocab, bnb non-determinism across GPU archs, rank saturation misread as quantization error.
- **Diagnosis ladder:** data → tokenizer → rank → LR/schedule → rehearsal mix → precision (last).

## Falsification Protocol

1. **NF4 vs fp16 LoRA A/B** on Qwen2.5-0.5B smoke (identical eval conditions).
2. **No-train base PPL** 4-bit vs fp16 on Hawaiian dev set.
3. **Rank ablation** r ∈ {16, 32, 64, 128} at fixed NF4 to distinguish capacity-limit from quant-limit failures.
4. **Long-train PPL curve** across provider switches to distinguish forgetting from quantization drift.
5. **Concede to fp16 LoRA only if:** Test 1 shows >2% relative gap AND Test 3 shows we're not rank-bound.
